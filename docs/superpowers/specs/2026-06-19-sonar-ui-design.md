# 声呐 Sonar Chrome 扩展界面设计

## 1. 原型来源与适配原则

本设计参考用户提供的三份原型：

- `Sonar 声呐.dc.html`：完整工作台及主要交互状态。
- `Sonar 插件适配.dc.html`：Popup、Side Panel、内容注入和通知。
- `Sonar 插件形态.dc.html`：当前为空，不作为功能依据。

原型决定产品的信息架构、视觉语言和主要交互。正式实现需要补齐原型尚未覆盖的无水印状态、清晰度选择、下载进度、AI 处理阶段、Provider 设置、错误与空状态。

当视觉原型与主设计文档的安全、数据或技术约束冲突时，以主设计文档为准。

## 2. 视觉语言

- 产品名称：声呐 Sonar。
- 主色：`#0a84ff`。
- 主界面：深色半透明面板、细描边、轻阴影和中等圆角。
- 正文：系统字体栈，优先 SF Pro 与苹方。
- 数值、时长和状态计数：随扩展本地打包的 JetBrains Mono 子集；加载失败时回退系统等宽字体。
- 状态色：蓝色表示主要操作，绿色表示健康监听，橙色表示等待或降级，红色表示错误或熔断。
- 动效仅用于同步旋转、处理中脉冲、Toast 入场和进度变化；支持 `prefers-reduced-motion`。

原型中的 macOS 窗口红黄绿按钮只属于展示稿。完整工作台实际运行在扩展 Tab 中，不伪造系统窗口控制。

## 3. 信息架构

第一期提供四个表面：

```text
Popup
  当前页面 / 新视频 / 快捷动作

Side Panel
  动态流 / 视频库 / 链接入库

抖音页面注入
  加入监听 / 视频入库 / 下载原片

完整工作台
  动态流 / 视频库 / 博主管理 / 工作流 / 设置
```

Popup 和 Side Panel 面向即时操作；完整工作台面向检索、阅读、处理和管理。相同对象状态必须来自统一 Repository，不能在不同表面维护独立副本。

## 4. Popup

### 4.1 尺寸与结构

- 固定宽度 380px。
- 顶部：Sonar 标识、新视频数量、刷新、设置。
- 当前页面卡片：页面类型、博主或视频信息、主要动作。
- 新视频区域：最多显示最近 3 条未读内容。
- 底部：打开完整工作台、打开 Side Panel。

### 4.2 当前页面状态

博主主页：

- 未监听：主要按钮为「监听」。
- 已监听：展示绿色监听状态，并提供「立即同步」。
- 无法识别：展示粘贴主页链接入口。

视频页或作品弹层：

- 展示标题、博主、时长和封面。
- 主要按钮为「下载原片」。
- 次要按钮为「入库并分析」。
- 若博主未监听，提供「同时监听博主」复选项。

下载按钮旁展示水印判断：

- 「高置信度无水印」使用绿色状态。
- 「可能无水印」使用橙色状态并可查看判断依据。
- 「仅找到带水印版本」默认不下载，要求用户明确确认。

### 4.3 快捷任务

任务进行时当前页面卡片切换为阶段进度：

```text
识别视频 -> 获取原片 -> 下载
                     -> 提取音频 -> 转录 -> 生成摘要
```

关闭 Popup 不取消任务。再次打开后从持久化任务状态恢复。

## 5. Side Panel

### 5.1 结构

- 建议宽度 360px，允许 Chrome 调整。
- 顶部：动态标题、刷新、关闭。
- Segmented Control：动态流 / 视频库。
- 上下文状态：当前站点、监听博主数量和熔断状态。
- 底部固定链接输入框：粘贴视频链接快速入库。

### 5.2 动态流卡片

卡片展示：

- 封面、时长和 NEW 标记。
- 博主、发布时间和标题。
- 点赞、评论、分类。
- AI 处理状态。

卡片操作：

- 点击正文：在完整工作台打开详情。
- 「下载」：解析并下载优先无水印源。
- 「分析」：入队转录和摘要。
- 「重点」：切换收藏标记。

### 5.3 视频库

视频库支持搜索和以下筛选：

- 全部
- 未读
- 重点
- 已归档
- 处理中
- 处理失败

侧边栏只提供轻量列表和快捷操作；全文字幕、Provider 错误和工作流编辑跳转完整工作台。

## 6. 抖音页面注入

### 6.1 博主主页

优先在主页操作区附近注入「加入声呐监听」按钮。注入 UI 使用 Shadow DOM 隔离样式；若找不到稳定锚点，则回退为页面右下角固定的 Sonar 浮动胶囊，不强行修改抖音布局。

状态包括：

- 加入监听
- 已监听
- 正在同步
- 登录失效
- 监控暂停

### 6.2 视频页和作品弹层

注入紧凑操作组：

- 下载原片
- 入库并分析
- 重点标记

下载前通过小型浮层显示质量、编码、水印置信度和预计大小。没有高置信度无水印源时，不能把主按钮文案继续显示为「下载无水印」。

## 7. 完整工作台

完整工作台通过扩展 URL 在普通 Tab 中打开。保持原型的深色桌面工作台观感，但使用浏览器页面语义，不渲染虚假的系统标题栏。

### 7.1 全局结构

- 顶部：当前模块、上下文摘要、全局搜索、全部同步、添加。
- 左侧：品牌、主导航、博主快捷列表、添加博主。
- 中部：当前列表。
- 右侧：视频详情或任务详情。

主导航：

- 动态流
- 视频库
- 博主管理
- 工作流
- 设置

### 7.2 动态流

- 支持全部博主或单博主过滤。
- 支持按时间、热度排序。
- 列表项展示封面、标题、作者、时间、互动指标、分类、未读和重点状态。
- 选中视频后自动标记为已读，并在右侧展示详情。

### 7.3 视频详情

详情头部：

- 封面、标题、博主、发布时间、原视频链接。
- 点赞、评论、收藏、分享、时长。
- 下载原片、重新分析、重点、导出摘要、加入工作流。

下载区域：

- 清晰度与编码候选。
- 水印判断、置信度和证据。
- 文件大小与下载进度。
- 过期重解析、取消和重试。

AI 摘要区域：

- 内容分类标签。
- 摘要正文。
- 关键要点。
- 话题标签。
- Provider、模型和生成时间。
- 重新生成摘要。

字幕区域：

- 时间轴片段与全文视图。
- 点击时间戳打开抖音原视频对应位置；无法精确跳转时至少打开原视频。
- 复制全文、下载 SRT、重新转录。
- 转录失败时保留视频详情和下载能力。

### 7.4 视频库

- 搜索标题、博主、标签、摘要和字幕。
- 筛选全部、未读、重点、已归档、处理中、失败。
- 支持多选下载、批量分析、批量导出 Markdown 和批量归档。
- 批量操作仍遵循单媒体处理队列，UI 显示队列位置。

### 7.5 博主管理

每个博主卡片展示：

- 昵称、抖音号、分组和监听状态。
- 视频数、未读数、最后同步时间。
- 立即同步、暂停/恢复监听、编辑备注和移除。

添加弹窗支持：

- 博主主页链接或抖音号。
- 备注名称与分组。
- 首次抓取数量，默认最近 20 条。
- 监控周期，默认 30 分钟。

### 7.6 工作流

第一期是三列本地看板：

- 待处理
- 处理中
- 已完成

工作流卡片引用视频，展示标题、博主、分类、备注和更新时间。支持移动状态、编辑备注、打开视频详情和移除。工作流不触发自动发布或外部系统调用。

### 7.7 设置

设置分组：

- 监听：周期、首次抓取数量、随机间隔、Chrome 启动补偿检查。
- 下载：目录模板、重复文件策略、无水印回退策略、默认质量。
- ASR：Provider、Base URL、API Key、模型、语言和测试。
- 摘要：OpenAI-compatible Base URL、API Key、模型、Temperature 和测试。
- 存储：数据库占用、临时缓存占用、清理缓存、导出或清空全部数据。
- 诊断：最近错误、解析器版本和脱敏日志导出。

## 8. 添加与入库

添加弹窗保留原型的双 Tab：

- 添加博主。
- 粘贴视频链接。

视频链接解析后展示确认页，而不是立即静默执行：

- 识别结果与目标博主。
- 是否监听该博主。
- 是否立即下载。
- 是否立即转录并生成摘要。
- 无水印回退策略。

添加博主时还需显示「自动分析新视频」和「分析最近作品」选项。用户未完成 Provider 配置与数据发送确认前，这两个选项默认关闭并解释原因。

提交后关闭弹窗并显示可追踪任务，不用 Toast 代替真实进度。

## 9. 通知

发现新视频时使用 Chrome 系统通知：

- 标题：`{博主} 发布了新视频`。
- 正文：视频标题。
- 主要按钮：查看详情。
- 次要按钮：下载原片。

点击通知正文在完整工作台打开对应视频。下载按钮仍需遵循无水印策略；若需要用户确认带水印回退，则先打开详情，不直接下载。

## 10. 状态与反馈

所有表面共享以下状态语言：

- 正常：已监听、已入库、已下载、分析完成。
- 进行中：同步、解析、获取媒体、提取音频、转录、摘要、下载。
- 等待：队列位置、Provider 未配置。
- 降级：仅找到低置信度或带水印源。
- 错误：登录失效、验证码、访问限制、地址过期、ASR 失败、摘要失败。

错误提示必须包含下一步动作，例如「重新登录抖音」「打开验证页面」「重新解析」「配置 ASR」「只重试摘要」。

## 11. 空状态

- 无监听博主：解释监听价值，提供当前页面识别和粘贴主页链接。
- 暂无新视频：展示上次同步时间和立即同步按钮。
- 视频库为空：提供粘贴链接与从当前页面入库。
- Provider 未配置：视频仍可下载，AI 区域显示配置入口。
- 工作流为空：解释如何从视频详情加入。

## 12. 可访问性与键盘操作

- 所有图标按钮提供可访问名称和 Tooltip。
- 颜色状态同时使用图标或文字，不仅依赖颜色。
- 焦点样式清晰，弹窗正确管理焦点与 Esc 关闭。
- 完整工作台支持 `/` 聚焦搜索、`J/K` 移动列表选择、`D` 下载、`E` 导出摘要；输入框聚焦时不触发快捷键。
- 正文与次要文字达到可读对比度，不能机械照搬原型中过低对比度的灰色。

## 13. UI 验收

- 四个表面展示同一对象的一致状态。
- 当前页面为博主页或视频页时，Popup 能显示正确的上下文操作。
- 下载前明确显示水印置信度；低置信度不冒充确定无水印。
- 关闭 Popup 或 Side Panel 后，下载与 AI 任务继续执行并可恢复进度。
- 完整工作台能完成监听、入库、下载、转录、摘要、导出和工作流的完整闭环。
- Provider 未配置或单一阶段失败不会阻断监听和下载。
- 验证码或登录失效在所有表面使用一致错误状态，并暂停自动监控。
